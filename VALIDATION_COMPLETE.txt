📊 STRUCTURE TECHNIQUE
================================================================================

Base de données (biometrie.db):
┌─ utilisateurs
│  ├─ id
│  ├─ nom
│  ├─ prenom
│  ├─ role
│  ├─ autorise (0/1)
│  └─ photo_path
│
└─ logs_reconnaissance
   ├─ id
   ├─ user_id (0 pour imposteur)
   ├─ nom
   ├─ statut (autorise/refuse/imposteur)
   ├─ confiance (0-200+)
   ├─ date_heure (TIMESTAMP)
   ├─ image_path (PNG file)
   └─ watermark_data (métadonnées encodées)

Fichiers PNG (logs/):
├─ log_20260503_152537_autorise.png (watermark LSB)
├─ log_20260503_152537_refuse.png (watermark LSB)
└─ log_20260503_152537_imposteur.png (watermark LSB)

🔄 FLUX DE TRAITEMENT COMPLET
================================================================================

1. Démarrage de l'application
   └─ init_db() crée les tables

2. Clic "Démarrer la reconnaissance"
   └─ RecognitionThread démarre
      └─ Boucle: capture WebCam (30ms/frame)

3. Chaque frame capturée:
   ├─ Détection des visages (Cascade)
   ├─ Prédiction LBPH
   ├─ Analyse du statut
   │  ├─ Si confiance < 60 + autorise=1 → "autorise"
   │  ├─ Si confiance < 60 + autorise=0 → "refuse"
   │  └─ Si confiance > 100 ou inconnu → "imposteur"
   └─ Appel à sauvegarder_log_tatatoue()

4. Sauvegarde du log:
   ├─ Création PNG avec watermark LSB
   ├─ Insertion en BD (logs_reconnaissance)
   └─ Affichage du statut dans l'UI

5. Clic "Historique"
   └─ page_historique.rafraichir()
      ├─ get_tous_logs() depuis la BD
      ├─ Formatage des données
      └─ Affichage du tableau + statistiques
